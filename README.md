# Smart License Plate Detection

> Real-time Indian license plate detection + OCR + database storage using a fine-tuned YOLOv5s and EasyOCR.

---

## 📌 Project Overview

**Smart License Plate Detection** is an end-to-end machine learning pipeline that:
1. **Detects** license plates in images, videos, or live webcam feeds using a **YOLOv5s** model fine-tuned on 3,500+ Indian plate images
2. **Reads** the plate text using **EasyOCR** with multi-variant preprocessing for higher accuracy on small/blurry plates
3. **Stores** every detection (plate number, confidence, timestamp, source) in a **SQLite database**

### Use Cases
- Traffic monitoring & surveillance
- Parking management systems
- Toll collection automation
- Law enforcement support

## ⚙️ Installation

### 1. Clone the Repository
```bash
git clone https://github.com/YASHzip/Smart-License-Plate-Detection.git
cd Smart-License-Plate-Detection
```

### 2. Install Dependencies
```bash
pip install -r requirements.txt
```

> **GPU Users (Recommended):** Install PyTorch with CUDA support for significantly faster inference:
> ```bash
> pip uninstall torch torchvision -y
> pip install torch torchvision --index-url https://download.pytorch.org/whl/cu128 --no-cache-dir
> ```
> Verify GPU is detected:
> ```bash
> python -c "import torch; print(torch.cuda.is_available(), torch.cuda.get_device_name(0))"
> ```

### 3. Download the Original Dataset (Optional — Baseline Model)
Download the dataset from Kaggle and place it in the `archive/` folder:

📎 [Kaggle Car Plate Detection Dataset](https://www.kaggle.com/datasets/andrewmvd/car-plate-detection)

### 4. Convert XML Annotations to YOLO Format
```bash
python xmltotxtconvert.py
```

## 🇮🇳 Indian Plate Fine-Tuning Workflow

This section documents the full workflow used to achieve **99.4% mAP** on Indian plates.

### Step 1 — Download a Multi-State Indian Dataset
```bash
# Recommended datasets (Kaggle):
# https://www.kaggle.com/datasets/theiturhs/indian-vehicle-number-plate-dataset
kaggle datasets download -d theiturhs/indian-vehicle-number-plate-dataset
Expand-Archive indian-vehicle-number-plate-dataset.zip -DestinationPath indian_raw
```

### Step 2 — Prepare the Merged Dataset
```bash
# Merges archive/ + new Indian data, auto-splits 80/10/10
python prepare_indian_dataset.py --new-dataset D:\indian_raw
```

### Step 3 — (Optional) Generate Synthetic Data
```bash
python generate_synthetic_data.py --plates D:\synthetic_plates --output synthetic_dataset --count 4000
python prepare_indian_dataset.py --new-dataset synthetic_dataset
```

### Step 4 — Fine-Tune the Model
```bash
cd yolov5
python train.py ^
  --img 640 ^
  --batch 8 ^
  --epochs 150 ^
  --data ..\indian_data.yaml ^
  --weights runs\train\exp2\weights\best.pt ^
  --hyp ..\hyp_indian.yaml ^
  --project runs\train ^
  --name indian_finetune ^
  --device 0
```

### Step 5 — Resume After Crash
```bash
# YOLOv5 saves last.pt every epoch — resume seamlessly:
python train.py --resume runs\train\indian_finetune6\weights\last.pt
```

### Step 6 — Validate
```bash
python val.py ^
  --weights runs\train\indian_finetune6\weights\best.pt ^
  --data ..\indian_data.yaml ^
  --img 640 ^
  --device 0
```

## 🔍 Usage Guide

### Testing – Quick Inference
```bash
python testing.py                                         # default test image
python testing.py --image path/to/image.png               # custom image
python testing.py --image img.jpg --conf 0.3 --no-save    # lower confidence
```

### Evaluation – mAP, Precision, Recall, F1
```bash
# Full evaluation on Indian test set
python evaluate.py --weights yolov5/runs/train/indian_finetune6/weights/best.pt --data indian_data.yaml --img 640

# Training summary only (faster)
python evaluate.py --weights yolov5/runs/train/indian_finetune6/weights/best.pt --skip-val
```
Output: `evaluation_report.txt` + annotated images in `evaluation_output/`

### OCR Pipeline – Read Plate Text
```bash
python ocr_pipeline.py --image plate_crop_test.jpg
python ocr_pipeline.py --image plate_crop_test.jpg --no-preprocess
```

### Full Pipeline – Detect + OCR + Save to DB
```bash
python detect_and_recognize.py --source archive/images/test/Cars255.png        # single image
python detect_and_recognize.py --source indian_dataset/images/val              # folder of images
python detect_and_recognize.py --source road_video.mp4                         # video
python detect_and_recognize.py --source 0                                      # webcam
```

**Useful flags:**

| Flag | Description |
|------|-------------|
| `--weights path` | Use a specific model weights file |
| `--img 640` | Set inference image size (default: 416) |
| `--no-save` | Don't save annotated output to disk |
| `--no-db` | Skip database saving |
| `--no-ocr` | Detection bounding boxes only |
| `--conf 0.5` | Set detection confidence threshold |
| `--show` | Display output window |

### Auto-Annotation
```bash
# Auto-label a folder of unlabelled images using the trained model
python auto_annotate.py --images D:\unlabelled_plates --output D:\labelled_output --min-conf 0.55 --copy-images
```

### Database Management
```bash
python database.py              # view all records
python database.py --search MH  # search by plate text
python database.py --count      # total record count
python database.py --clear      # delete all records
```

**Database Schema (`plates.db → detections`):**

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Auto-incremented primary key |
| `plate_number` | TEXT | Cleaned plate text (e.g. `MH12AB1234`) |
| `raw_ocr_text` | TEXT | Raw EasyOCR output |
| `image_path` | TEXT | Source file path |
| `detection_confidence` | REAL | YOLOv5 bounding box confidence |
| `ocr_confidence` | REAL | EasyOCR average confidence |
| `source` | TEXT | `image`, `video`, or `webcam` |
| `timestamp` | TEXT | Detection datetime (`YYYY-MM-DD HH:MM:SS`) |
